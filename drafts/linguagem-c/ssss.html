OEPRAÇÃO TERNÁRIA

O Operador Ternário possui uma estrutura onde definimos um Expressão Lógica e dois valores. O primeiro será retornado, caso a expressão seja verdadeira, do contrário o segundo valor será atribuido.

Uma Operação Ternária pode ser escrita utilizando a Tomada de Decisão, porém, enquanto a instrução if utiliza no mínimo 4 linhas de código, a Operação Ternária utiliza uma única linha, simplificando assim o código e melhorando a legibilidade;

A seguir, temos um exemplo utilizando o Operador Ternário.

#include <stdio.h>
#include <stdlib.h>

int main()
{
    printf("Digite uma letra: ");
    char caractere = 0;
    scanf( "%c", &caractere );

    int num = (caractere=='a') ? 65 : 66;

    printf("O códino ASCII da letra digitada é: %i", num);

    return 0;

}

No código a seguir, há somente uma linha que foi extraída do cóidgo acima para que possamos estudar em detalhes o Operador Ternário.

int num = (caractere=='a') ?   65   :   66;
<valor> =        < x >     ? (True) : (False)

Inicialmente, estamos declarando uma variável de nome num do tipo int e estamos adicionando a mesma o valor resultante da resolução da estrutura do Operador Ternário.

O sinal de interrogação deve ser lido como: "A expressão é verdadeira?", ou seja, como o sinal denota, de fato estamos questionando o resultado da expressão lógica. Se o Valor Lógico for True o número 65 será retornado, senão, ou seja, o sinal de dois pontos : está substituíndo a instrução else. Logo, devemos ler "senão", retorna o segundo valor definido na estrutura da Operação Ternária.

Por fim, temos que ler a expressão da seguinte maneira: O Valor Lógico da expressão é verdadeiro? Se sim, retorna o primeiro valor : senão, retorna o segundo valor. A seguir, vamos temos dois códigos que estão realizando a mesma tarefa, porém, um está escrito com a instrução if, enquanto o segundo, com o Operador Ternário.

//utilizando o Operador Ternário
int num = (caractere=='a') ? 65 : 66;

//utilizando a instrução "if"
if (caractere=='a')
    num = 65
else
    num = 66;

Por fim, se você compreendeu o uso da Tomada de Decisão, ou seja, da instrução if, conseguirás assimilar rapidamente essa notação. Até porque, a lógica de funcionamento é a mesma, a diferença está no "esquema" de cada estrutura.
