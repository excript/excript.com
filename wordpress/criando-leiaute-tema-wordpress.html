DESENVOLVIMENTO DE TEMAS PARA WORDPRESS

Publicado em 01 January 2016 por Cláudio Rogério Carvalho Filho
  • ÍNDICE
  • ARTIGO
  • VIDEOAULA
NOME DURAÇÃO

INTRODUÇÃO

O Wordpress disponibiliza uma API para que possamos desenvolver os nossos próprios temas. Tudo que precisamos para desenvover qualquer tipo de tema está contido em qualquer instação do Wordpress.

Num primeiro momento, vamos estudar a estrutura fundamental da API para desenvolvimento de temas para o Wordpress e em seguida, começaremos a adicionar o HTML e o CSS junto ao código PHP.

ESTRUTURA DE ARQUIVOS E DIRETÓRIO

Para que possamos desenvover plugins e temas para o Wordpress não é necessário que conheçamos a estrutura de pastas, os arquivos PHP e seus códigos. Isso porque, tanto no desenvolvimento de um simples tema até o desenvolvimento de um plugin complexo, a interação com a instalação do Wordpress se dará através da API disponibilizada para este fim.

Ainda assim, vale a pena entender-mos a estrutura, seus principais arquivos, funções e classes. Por isso, agora, estudaremos rapidamente o que há de mais importante na estrutura de diretórios e alguns arquivos do Wordpress.

ESTRUTURA

Abaixo temos a estrutura de diretórios de uma instalação do Wordpress. A raiz é "wordpress" mas poderia ser qualquer outro nome, porém, os arquivos que estão contidos dentro do diretório raíza não podem ter seus nomes alterados.

É importante observar que a nomenclatura dos diretórios e pastas é case-sensitive, ou seja, a letra "A" maiúscula é diferente da letra "a" minúscula.

-wordpress/
├── wp-admin/ ├── wp-content/ ├── wp-includes/ ├── ├── ... ├── ├── .htaccess ├── index.php ├── wp-blog-header.php

DIRETÓRIO wp-admin

O diretório wp-admin contém os arquivos do dashboard do Wordpress, ou seja, nesta pasta nós temos a programação do back-end. Todas as funções administrativas ou então, qualquer função que esteja relacionada com o Wordpress, criação de páginas, criação de postagens, seleção de qual tema será utilizado, os widges dentre outros, estão contidos dentro desse diretório.

Salvo se você estiver criando um fork (versão própria) do Wordpress é que necessitaras alterar o código de algum arquivo contido no diretório "wp-admin". E também, é altamente desancolhavel que você faça alterações, até porque, quando fores atualizar o framework, os arquivos que havias alterados serão sobreescritos e assim, perderas o seu código e provavelmente, a sua instalação do Wordpress deixara de funcionar. Então, salvo em casos especiais é que iremos fazer alterações na pasta "wp-adim", porém, é super recomendável que você estude e entenda o funcionamento.

DIRETÓRIO wp_includes

Neste diretório estão contidos o código do núcleo do Wordpress (core). Então, a maior parte das classes e funções que trabalhamos estão definidas em arquivos contidos neste diretório.

DIRETÓRIO wp_content

O diretório wp_content contém os temas disponíveis, os plugins baixados e as imagens que foram enviadas. Então, "wp_content" faz juz a seu nome, até porque, é a pasta que contém uma parte do conteúdo das nossas páginas. (Os textos, descrições, títuloes e etc são armazenados dentro do banco de dados, então, você não encontrará esse tipo de informação me qualquer diretório ou subdiretório de uma instalação do wordpress).

Dentro do diretório wp_content, temos outros 3 subdiretórios, são eles:

-wp-content/
├── languages/ ├── themes/ ├── plugins/ ├── upgrade/ ├── uploads/

DIRETÓRIO themes

O diretório "theme" contém todos os temas que estão instalados em nosso Wordpress. Por padrão, quando baixamos o arquivo compactado do site do wordpress.org - http://wordpress.org, o mesmo vem com 3 temas, são eles:

twentyfifteen twentyfourteen twentythirteen

Esses 3 temas que que vem por padrão tem o objetivo de orientar o desenvolvimento de temas, e por isso, os mesmo são bastante simples - para que consigamos entender facilmente. É comum programadores personalizarem seus temas partindo de um desses 3 modelos.

Então, até agora, a informação mais importante para o nosso estudo é saber que todos os temas instalados estão contidos no diretório "wp-content/themes/nome-do-tema". A segunda, é que TODO tema está contido dentro de uma única pasta.

Numa instalação do Wordpress, podemos ter instalados quantos temas desejarmos, porém, só é possível ativarmos um tema por vez. Toda instalação do Wordpress em funcionamento, terá, obrigatoriamente um tema, ou seja, se acessarmos o nosso site e não houver um tema disponivel, sera levantado uma exceção.

DIRETÓRIO plugins

O diretórios plugins possui um funcionamento semelhante ao diretório theme, porém, é possível, ainda que pouco provável, rodarmos uma instancia do Wordpress sem qualque Plugin ativado ou mesmo, instalado.

Assim, como o nome do diretório sugere, todos os plugin que estiverem instalados numa instância do Wordpress, estará, obrigatoriamente, contido no diretório plugins.

DIRETÓRIO uploads

Todos os arquivos que fizermos upload será armazenado por padrão, no diretório "wp-content/uploads". Dentro do diretório uploads será criado um subdiretório para cada ano. Dentro deste, haverá subdiretórios para cada mês e dentro deste, haverá subdiretório para cada dia em que enviamos algum arquivo para o servidor.

O nome do arquivo que enviamos será preservado. Caso o arquivo enviado seja por exemplo, uma imagem, o Wordpress irá criar 2 outras imagens menores, também chamadas de "thumbnails". Estas seguirão a nomenclatura:

  • nome_do_arquivo.png
  • nome_do_arquivo-150x150.png
  • nome_do_arquivo-300x174.png

Então, em cada situação o Wordpress irá abrir um determinado tipo de arquivo, a fim de melhor servir os usuários.

DIRETÓRIO upgrade

O diretório upgrade será utilizado todas as vezes em que formos atualizar a instalação do Wordpress. A pasta "upgrade", será utilizada para o armazenamento temporário de diversos arquivos, dentre eles, o download da nova versão do Wordpress, como também, o respectivo backup que será realizado antes que a atualização seja processada.
