BOOTSTRAP E WORDPRESS

DESENVOLVIMENTO DE TEMAS PARA WORDPRESS

Publicado em 01 janeiro 2016 por Cláudio Rogério Carvalho Filho

NOME DURAÇÃO

INTRODUÇÃO

Bootstrap é um framework open-source para desenvolvimento frontend, desenvolvido pelo Twitter cujo objetivo é simplificar a construção de páginas web.

A construção foi iniciada por Mark Otto e Jacob Thornton em meados de 2010 e inicialmente foi chamado de Twitter Blueprint. A meta inicial foi desenvolver uma ferramenta interna para a padronização dos serviços da empresa twitter.com. Isso porque, diferentes grupos adotavam diferentes frameworks e bibliocas fazendo com que os diversos serviços tivessem visuais diferentes e o pior, sem qualquer tipo de padronização do código.

A primeira versão pública do framework foi lançada em 19 de agosto de 2011 e recebeu o nome de Twitter Bootstrap.

RECURSOS

O framework Bootstrap oferece um conjunto de CSS e estes, definem desde a tipografia, até complexos campos de entrada. Logo, nós temos que ao utilizar o framework, este irá aplicar um padrão à exibição de textos, tabelas, formulários, botões, e os principais controles que são utilizados na construção de uma aplicação web e/ou websites.

MOBILE-FIRST

Mobile-First é a técnica onde desenvolve-se o leiaute inicialmente para dispositivos Mobile e em seguida, para as outras resoluções de tela.

O padrão para o desenvolvimento de leiautes do Bootstrap é voltado para dispositivos móveis ou então, para dispositivos que possuem uma pequena tela. Em seguida, desenvolve-se para tablets, ou então, dispositivos que tenham uma tela de tamanho mediano e por fim, da-se o desenvolvimento para telas que possuem maior resolução. Essa característica é chamada de "Mobile First", ou seja, inicialmente desenvolve-se para Mobile e em seguida para outras resoluções.

DEPENDENCIAS

O Bootstrap utiliza somente a biblioteca JQuery. Então, todas as vezes em que formos incluir o Bootstrap em nossa HTML, temos que antes, incluir a biblioteca JQuery.

A METAINFORMAÇÃO viewport

<meta name="viewport" content="width=device-width, initial-scale=1">

A viewport que é definida por padrão nas demonstrações do Bootstrap, definem que a largura inicial do documento, será a largura do dispositivo em que está sendo exibido width=device-width. O valor da propriedade width do viewport pode ser alterada, porém, o framework Bootstrap também cuidará dessa definição pra gente.

Em seguida, nós definimos que a escala de zoom do documento será igual a 1, ou seja, a página será exibida na proporção de 1 para 1 initial-scale=1.

É possível impedir que o usuário dê zoom na página, porém, na maior parte das vezes não é interessante definirmos essa propriedade e até a documentação do Bootstrap diz para os desenvolvedores agirem com cautela. Para impedir o zoom, basta adicionarmos na viewport a instrução user-scalable=no, logo, a visualização não será escalável, ou seja, não será possível aumentar e diminuir o tamanho geral da página.

CLASSES CSS ESTRUTURAIS

O Twitter Bootstrap definiu 4 classes CSS como sendo estruturais, ou seja, elas não possuem uma visualização propriamente dita, porém, estas irão definir como que outras tags serão exibidas.

Estas classes podem ser utilizadas com qualquer tag HTML, porém, as mesmas geralmente são utilizadas em TAGS div.

A seguir, temos a lista das 4 classes estruturais:

1 - .container 2 - .container-fluid 3 - .row 4 - .col

A classe .col é constituida pelo conjutno de 48 classes a fim de definir as 4 dimensões reconhecidas pelo framework e a quantidade de unidades que cada classe irá ocupar.

A CLASSE .container

A classe .container é a principal de todo documento e a mesma sempre estará definida na hierarquia mais alta da arvore de tags HTML. O objetivo de um container é ser o ponto inicial da construção dos documentos. Assim, os scripts do framework irão utilizar o mesmo como sendo a base de construção.

Todo documento que irá utilizar o framework Twitter Bootstrap deverá, obrigatoriamente, ter no mínimo uma tag com a classe container. Porém, o comum é que os documentos utilizem 3 vezes essa classe. Uma para definir o cabeçalho, outra para o corpo e uma última para definir o rodapé do documento.

O framework não obriga a utilização de 3 tags num documento a fim de definir as 3 áreas comuns a maioria das págians Web, assim, essa é uma boa prática de desenvolvimento e que deve ser alterada, conforme a necessidade do projeto.

A CLASSE .row

A classe .row tem o objetivo de definir um segmento horizontal, isto é, definir uma área que seja capaz de receber componentes que somados utilizem 12 unidades. Então, é comum que a classe .row seja a primeira filha da classe .container e também, é comum que dentro desta, haja tags com a classe .col.

Assim, a estrutura básica de um documento será sempre um container, que contém uma ou mais linhas (row) e dentro desta, haverá a definição propriamente dita do sistema de 12 colunas.

A CLASSE .col

A classe .col é constituida por um conjunto de 48 classes e representam o núcleo do sistema 12 colunas. É comum encontrarmos colunas dentro de linhas .row, até porque, essa é a estrutura recomendada pelos desenvolvedores do framework.

Assim, podemos imaginar que a classe .row representa um segmento horizontal, enquanto que as classes .col dividem esse segmento a fim de tornar o leiate ajustável às varias resoluções de tela.

ALGUNS CONCEITOS IMPORTANTES SOBRE O BOOTSTRAP

Algumas ideias que devem ser reforçadas quando iremos trabalhar com o framework Bootstrap.

  1. A classe container e a classe container-fluid não possuem diferença quanto a função de servirem como um recipiente que recebe outros objetos. A diferença maior acaba sendo visual, até porque, o container-fluid ocupara toda área disponível.
  2. Todo segmento horizontal (row), como também, qualquer outro componente, deve estar, obrigatoriamente, dentro de um container. Até porque, todos os cálculos para o redimensionamento se baseara nos mesmo.
  3. Colunas devem ser filhas de linhas (row), então, podemos pensar que uma row é um container exclusivo para os objetos colunas. Indiferente do conteúdo que será exibido numa linha, estes nunca devem estar contidos na row propriamente dita, até porque, somente colunas devem ser filhas da classe row.
  4. Resoluções de tela superior a 1200 exibiram o conteúdo conforme definido pela class lg ou então, que foi definida numa resolução abaixo da mesma.
