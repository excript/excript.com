TEMPLATE HIERARCHY

Definição da ordem utilizada pelo Wordpress para decidir qual Template file será aberto e processado para renderizar determinado conteúdo. O Wordpress definiu uma estrutura especializada que será avaliada antes de definir qual template utilizar para os diversos tipos de conteúdos. O nível mais baixo da hierarquia é o Template file index.php.

VISÃO GERAL DA UTILIZAÇÃO DE TEMPLATES E TAGS

index.php

Este é o template file mais importante e o único obrigatório para executarmos um tema. Indifente do conteúdo ou programação contida no index.php este será o ponto de entrada na ausência de outros templates. É importante entender que a função principal do template index.php não é exibir a lista de postagens, mas sim, servir como template de reserva na ausência de algum outro template.

style.css

Arquivo necessário para que o tema funcione, até porque, é dentro do style.css que colocamos as informações sobre o nosso tema, como por exemplo, nome do tema, autor, versão e etc.

rtl.css

The right-to-left stylesheet é incluido automaticamente no sitge caso a direção do texto seja da direita para a esquerda.

comments.php

Este é o template file onde iremos definir a lista de comentários de uma página ou postagem.

front-page.php

Este é o pronto de entrada da raiz de todo website Wordpress. Porém, o template front-page.php só será invocado, se e somente se, a configuração em Admin > Configurações > Leitura estiver definindo uma página estática. Ou seja, o template front-page.php tem o objetivo de utilizar uma página como sendo a frente do website, enquanto que o template home.php tem o objetivo de ser o ponto de entrada do blog, mostrando assim, a lista dos últimos posts.

home.php

Template que exibe a lista das últimas postagens do blog. Este template deve ser entendindo como o ponto de entrada do blog. Mesmo que o template index.php venha a desempenhar a mesma função e até mesmo, possua o mesmo código que o template home.php, a função principal do index.php é servir como fallback, ou seja, como o template de reserva caso seja necessário renderizar um template que não esteja definido (programado).

É importante observar que este template só é invocado quando o template front-page.php não estiver definido ou então, quando não houver uma página estática definida em Admin > Configurações > Leitura.

header.php

s``Template file`` que contém o cabeçalho do documento HTML do nosso site. Neste arquivo, é onde colocaremos as meta tags do HTML, bem como os assets que devem ser abertos no carregamento da página.

single.php

Template que será invocado quando for necessário renderizar uma postagem em especifico.

single-{post-type}.php

Template especializado e que tem precedência ao template single.php. É neste lugar onde iremos programar, por exemplo, uma postagem que é do tipo imagem, ou book, link e etc. Então, este é um template especializado em que podemos personalizar as postagens.

archive-{post-type}.php

Este templape será utilizado nas solicitações para exibição, por exemplo, de uma imagem ou então, de algum outro arquivo que disponibilizamos através do Wordpress. Caso este template não esteja definido, o Wordpress irá utilizar o template genérico archive.php.

page.php

Utilizado para renderizar a exibição de uma Página.

page-{slug}.php

O Wordpress irá inicialmente, tentar renderizar um conteúdo de uma página especifica com o template especifico, que podemos definir através deste template. Caso o mesmo não exista, a página será renderizada com o template de página genérico.

category.php

O template category é utilizado quando é necessário exibir as postagens por categorias.

tag.php

O template tag é utilizado quando é necessário exibir as postagens por uma determinada tag.
