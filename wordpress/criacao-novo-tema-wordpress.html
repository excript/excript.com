NOME DURAÇÃO

CRIANDO UM NOVO TEMA

Há diversas maneira para iniciarmos o desenvolvimento de um tema para o Wordpress. Fato é, que não existe maneira correta, porém, existem pequenas bibliotecas ou então, conjuntos de arquivos que podemos utilizar para inicar os desenvolvimento.

Também é comum, desenvolvermos um tema a partir de outro, assim, ao invés de ter que ir adicionando as chamadas à API ou então, criar cada arquivo referente a cada parte do tema, podemos utilizar qualquer outro plugin.

Uma forma rápida para implementarmos um novo tema, é utilizarmos um dos 3 temas que estão contidos no diretório theme da versão de download do wordpress.org.

Nesse momento, o nosso objetivo não é somente desenvolver um tema, mas sim, entender e estudar a API como também, tomarmos conhecimentos dos principais nomes de métodos, funções e variáveis.

CRIANDO OS ARQUIVOS

Para iniciarmos o trabalho, vá paraa pasta wordopress/wp-theme e nesta, crie uma nova pasta. De preferência, utilize o nome do tema como nome da pasta. Essa é sem dúvida a melhor maneira de organizarmos o nosso códigos e também, distinguir entre os vários temas que em breve teremos.

Para o nosso estudo, vamos criar a pasta de nome wpstrap. Certifique-se de ter criado a pasta no mesmo local do path a seguir: wordopress/wp-theme/wpstrap.

Agora, vamos criar 2 arquivos. O primeiro, será a nossa folha de estilo, de nome style.css e o segundo, o arquivo de entrada para a renderizaçaõ do nosso tema, o arquivo index.php.

Agora, as 4 primeiras definições que temos de entender, assimilar e gravar.
  1. Todo tema está contido dentro de uma pasta
  2. Na raíz de todo tema haverá um arquivo CSS
  3. Dentro do arquivo CSS é que iremos definir as propriedades do nosso tema, como por exemplo, o seu nome.
  4. O tema será inicializado com a importação do arquivo index.php, por isso, esse arquivo também é obrigatório.

DEFININDO O style.css

Cada tema do Wordpress possui, obrigatoriamente, um arquivo de nome style.css. Não é necessário nenhum código CSS, porém, será através deste arquivo que o nosso tema será reconhecido dentro da instalação do dashboard do Wordpress.

Dentro do style.css, nas primeiras linhas, vamos que declarar as propriedades do nosso tema, como temos no código abaixo:

/*
Theme Name: PreXcript
Theme URI: http://excript.com
Author: Cláudio eXcript (@eXcript)
Author URI: http://excript.com/
Description: Estudo dirigido à criação e desenvolvimento de temas.
Version: 1.0
License: GNU General Public License v2 or later
License URI: http://www.gnu.org/licenses/gpl-2.0.html
Tags: eXcript
Text Domain: eXcript
*/

LISTA DAS TAGS A SEREM DEFINIDAS NO style.css

  • Theme name: Nome do tema
  • Theme URI: Endereço do site do desenvolvedor
  • Author: Nome do autor do tema ou da empresa
  • Author URI: Endereço do site do autor ou empresa
  • Description: Descrição breve e objetiva
  • Version: Versão do tema
  • License: A lincença na qual o tema está sendo disponibilizado
  • License URI: Endereço para as regras da licença
  • Tags: Lista de tags para ser utilizado quando o tema for disponibilizado no site oficial
  • Text Domain: String que será utilizada no momento em que fores disponibilizar o tema em outras versões. Normalmente, utilizamos o apelido que utilizamos quando enviamos o tema para o repositorio wordpress.org

TESTANDO O NOVO TEMA

Após criamos o arquivo style.css e index.php dentro da pasta do nosso tema e também, definirmos o nome e outros atributos do nosso tema dentro do CSS, já podemos testar o nosso tema. Então, acesse o Dashboard, clique em Aparência e depois Tema. Se você executos os passos anteriores corretamente, o nosso tema já estara aparecendo na parte administrativa da instalação do Wordpress.

ATIVANDO O NOVO TEMA

Para que possamos desenvolver e ao mesmo tempo ir testando no navegador, temos que ativar o nosso tema dentro do Dashboard. Então, como via de regra, esses sempre serão os passos iniciais que teremos de dar para criar um novo tema. Agora, ative o tema que estamos criando para que as próximas vezes em que acessarmos o Wordpress, o nosso tema seja exibido com as últimas modificações.

DEFININDO UMA IMAGEM PARA O TEMA

O nosso tema já está sendo exibido na tela de temas do Dashboard, porém, nenhuma imagem é exibida. Para definirmos a imagem que deve ser exibida, ou seja, a cara do nosso tema, temos que colocar a imagem na raiz do directorio do diretório do tema com o nome de screenshot.png e com as dimensões de 880x660.

Ainda que não seja obrigatório que a imagem seja do tipo PNG, na documentação do Wordpress, esse é o formato sugerido. Com a resolução da imagem ocorre o mesmo, não é obrigatório que a mesma deva possuir as dimensões 880x660, porém, essas são as dimensões recomendadas pela documetação oficial.
